HTTP Basics (RWA01)

Complete these tasks using browser DevTools (F12) and the terminal. No coding - just exploration.

Task 1: Anatomy of a URL
Break the following URL into its parts: scheme, host, port, path, query string, fragment.
https://www.example.com:8080/products/search?category=books&sort=price#results
Task 2: HTTP request/response in DevTools
Open https://jsonplaceholder.typicode.com/posts/1 in your browser. Open DevTools → Network tab, refresh the page and find the request. Identify:
  • The HTTP method
  • The status code and message
  • The Content-Type header from the response
  • The response size (Size column)
Task 3: curl from the terminal
Run in your terminal:
curl -v https://jsonplaceholder.typicode.com/posts/1

Identify in the output:
  • The request line (method, path, protocol version)
  • The response status line
  • Where do the headers end and the body begin?
Task 4: HTTP status codes
Visit each of the following URLs and note the status code (use the Network tab):
  • https://httpstat.us/200
  • https://httpstat.us/301
  • https://httpstat.us/404
  • https://httpstat.us/500
For each code, explain what it means.
Task 5: HTTP is stateless
Run these two commands in your terminal, one after the other:

curl -X POST https://httpbin.org/post -d "name=Alice"
curl https://httpbin.org/get

Does the second response contain any trace of "name=Alice"? Why not? What does this tell you about how HTTP works?
